## Overview

> <Badge text="wss" variant="tip" size="large" />
> <Badge text="message_read" size="large" />

The `message_read` request marks messages as read on the server and notifies other online users about this.

## Request Parameters

| Name  | Type            | Description                                |
| ----- | --------------- | ------------------------------------------ |
| `cid` | `string`        | Conversation ID where messages are located |
| `ids` | `array[string]` | Array of messages that need to be marked   |

<Aside>If `ids` is omit, we mark all unread messages as read.</Aside>

## Response Fields

<Tabs>
  <TabItem label="Response">
```json
{
  "id": "RequestId",
  "success": true
}
```
  </TabItem>
</Tabs>

## Example

<Tabs>
  <TabItem label="Request">
```json
{
  "request": {
    "message_read": {
      "cid": "63077ad836b78c3d82af0812",
      "ids": ["63480e68f4794709f802a2fa", "63077ad836b78c3d82af0866"]
    },
    "id": "RequestId"
  }
}
```
  </TabItem>
</Tabs>

<Aside type="caution">Conversation type `c` do not support read status.</Aside>

Then, all the users whose messages we read will receive the following message:

<Tabs>
  <TabItem label="Event">
```json
{
  "message_read": {

    "from": "634ec51c0b65918393dca5bf"
  }
}
```
  </TabItem>  
</Tabs>

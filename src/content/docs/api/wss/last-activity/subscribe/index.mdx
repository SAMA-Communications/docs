## Description

> <Badge text="wss" variant="tip" size="large" />
> <Badge text="user_last_activity_subscribe" size="large" />

The `user_last_activity_subscribe` request will allow you to subscribe to another user in order to track their activity status in real time.

## Request Parameters

| Name | Type     | Description |
| ---- | -------- | ----------- |
| `id` | `string` | User ID     |

<Aside type="caution" title="Important">
  You can only have one subscription, which means that the next one will cancel
  the previous one.
</Aside>

## Response Fields

| Name            | Type     | Description               |
| --------------- | -------- | ------------------------- |
| `last_activity` | `object` | Last user activity by UID |

## Example

<Tabs>
  <TabItem label="Request">
```json
{
  "request": {
    "user_last_activity_subscribe": {
      "id": "UID1"
    },
    "id": "RequestId"
  }
}
```
  </TabItem>  
</Tabs>

<Tabs>
  <TabItem label="Response">
```json
{
  "response": {
    "id": "RequestId",
    "last_activity": {
      "UID1": 0 | <last_activity-time>
    }
  }
}
```
  </TabItem>  
</Tabs>

Once that user will go offline/online, you will be receiving his/her last activity status:

<Tabs>
  <TabItem label="Event">
```json
"last_activity": {
  "UID1":  0 | num_of_seconds
}
```
  </TabItem>  
</Tabs>
